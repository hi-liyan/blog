## HTTP 诞生背景

1. CERN（欧洲核子研究组织）的蒂姆 • 伯纳斯 - 李（Tim BernersLee）博士提出了一种能让远隔两地的研究者们共享知识的设想。

2. 1990 年 11 月，CERN 成功研发了世界上第一台 Web 服务器和 Web 浏览器。两年后的 1992 年 9 月，日本第一个网站的主页上线了。

3. HTTP/0.9（HTTP 于 1990 年问世。那时的 HTTP 并没有作为正式的标准被建立。）

4. HTTP/1.0（HTTP 正式作为标准被公布是在 1996 年的 5 月，版本被命名为HTTP/1.0）

5. HTTP/1.1（1997 年 1 月公布的 HTTP/1.1 是目前主流的 HTTP 协议版本）

## 网络基础 TCP/IP 协议族

1. 通常使用的网络（包括互联网）是在 TCP/IP 协议族的基础上运作的。而 HTTP 属于它内部的一个子集。

2. TCP/IP 协议的分层设计，应用层、传输层、网络层、数据链路层。

3. 应用层决定了向用户提供应用服务时通信的活动，如HTTP、FTP、DNS等。

4. 传输层提供了网络连接中两台计算机之间的数据传输，传输层有两种协议 TCP （传输控制协议）和 UDP（用户数据协议）。

5. 网络层用来处理在网络上流动的数据包。数据包是网络传输的最小数据单位。该层规定了通过怎样的路径（所谓的传输路线）到达对方计算机，并把数据包传送给对方。

6. 数据链路层用来处理连接网络的硬件部分。包括控制操作系统、硬件的设备驱动、NIC（Network Interface Card，网络适配器，即网卡），及光纤等物理可见部分（还包括连接器等一切传输媒介）。硬件上的范畴均在链路层的作用范围之内。

7. 确保连接可靠性的三次握手：发送端先发出带有 SYN 标志的数据包给目标，对方成功收到数据包之后响应一个 SYN/ACK 数据包，发送端收到对方发来的 SYN/ACK 表示链路是通的，之后再发出一个应答 ACK 数据包，告诉对方我成功收到了，三次握手成功后建立连接。

8. URI（统一资源标识符） URL（统一资源定位符）
