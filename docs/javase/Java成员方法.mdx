方法是描述类的具体行为，提供操作数据的方式。

## 定义方法

编写Java代码时，时时刻刻都在和方法打交道，编写各种方法。所以，如何定义方法是很重要的。下面是一个简单的方法定义：

```java
public void hello() {
    System.out.println("Hello Java");
}
```

方法中的第一个位置用来指定方法的访问权限，如果没有指定访问修饰符，默认为`default`，这里声明为`public`；

第二个位置用来指定方法的返回值类型，如果方法没有返回值，使用`void`关键字；

第三个位置表示方法的名称，遵循标识符的命名规则，使用小驼峰命名，且要见名知意；

小括号`()`表示方法的参数列表；

后面的花括号`{}`表示方法体，是方法要执行的代码。

## 访问权限

|访问修饰符|类内|包内|子类|其它包|
|---|---|---|---|---|
|public|✅|✅|✅|✅|
|protected|✅|✅|✅|
|default|✅|✅|
|private|✅|

注意：`default`比较特殊，如果没有显式指定访问修饰符的话，访问权限就是default。

## 形参和实参

方法参数列表中定义的参数叫做**形式参数**，在调用方法传进来的参数叫做**实际参数**。

## 方法调用

在Java语言中，使用点标记法来调用方法。

```java
Cat cat = new Cat("花花");
cat.eat("猫粮");
```

## 按值传递

在Java语言中，方法参数的传递方式使用的是**按值传递**的方式。也就是说，在方法中，是不能通过形参来修改方法外实参的值的。这有别于其他一些按引用传递的编程语言。

这里也有一个小坑，如果实参是基本数据类型，那很容易就理解上面这句话。但如果实参是引用数据类型，在执行结果上好像与上面的描述不相符。但实际上，引用数据类型的参数也是按值传递的，因为引用类型的变量中存储的值实际上是内存的引用。将引用类型传给形参后形参指向的也是相同内存的对象，所以可以操作对象中的属性或者方法。但是在方法中，却不能改变实参的内存引用，即便在方法中让形参指向了一个新对象，但实际参数指向的仍然是原来的内存引用，这就是为什么说Java是按值传递的，在方法中不能通过形参来修改方法外实参的值。
